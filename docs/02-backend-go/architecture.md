## 📋 Overview

Operational Services (OS) trong Arda được xây dựng bằng ngôn ngữ **Go**, tập trung vào tốc độ thực thi cao, footprint memory thấp và khả năng scale linh hoạt.

## 🏗️ Technical Stack

- **Framework**: [Kratos v2.9](https://go-kratos.dev/)
- **Communication**: gRPC (Internal), HTTP/JSON (External via APISIX)
- **Dependency Injection**: [Google Wire](github.com/google/wire)
- **Database Driver**: [pgx v5](github.com/jackc/pgx)
- **Configuration**: Proto-based config (YAML/Environment)

## 🗂️ Project Structure (Monorepo)

```
arda-be/
├── api/                     # Shared Protobuf definitions
│   └── [service]/v1/*.proto
├── pkg/                     # Shared libraries (Auth, DB, Middleware)
└── [service-name]/          # Individual Microservice
    ├── cmd/                 # Entry point & Wire injection
    ├── internal/
    │   ├── biz/             # Domain layer (Business logic)
    │   ├── data/            # Data layer (Repo implementation)
    │   ├── service/         # Service layer (DTO mapping)
    │   └── conf/            # Config structs (Generated)
    ├── configs/             # YAML configs
    └── Makefile             # Build & Generate commands
```

## 🛡️ Key Patterns

### 1. Domain-Driven Design (DDD)
- **Biz Layer**: Chứa các interface Repository và Business Logic thuần túy. Không phụ thuộc vào database cụ thể.
- **Data Layer**: Implement các interface từ Biz layer, sử dụng `pgx` hoặc `redis`.

### 2. Multi-Tenancy (Row-Level Security)
Hệ thống sử dụng **PostgreSQL RLS** để cách ly dữ liệu:
- Middleware trích xuất `tenant_id` từ gRPC Metadata hoặc HTTP Header.
- `tenant_id` được truyền xuống tầng Data.
- Repository thực hiện `SET LOCAL app.current_tenant_id = '...'` trong mỗi transaction.

### 3. Event-Driven & Outbox Pattern
Đảm bảo tính nhất quán dữ liệu khi gửi message:
- Thay vì gửi trực tiếp tới Redpanda, service ghi message vào bảng `outbox` trong cùng transaction với nghiệp vụ.
- Một relay worker (hoặc Debezium) sẽ đọc bảng `outbox` và push vào Redpanda.

### 4. Forward Auth Architecture
- Các service không tự xác thực người dùng.
- APISIX gọi tới `iam-service` (Forward Auth) để kiểm tra JWT.
- Thông tin User & Roles được gắn vào Header để các service phía sau sử dụng.

## 🚀 Development Workflow

1. **API First**: Định nghĩa service interface trong `.proto`.
2. **Generate**: Chạy `make api` để generate gRPC/HTTP code.
3. **Implement**: Viết logic trong `internal/biz` và `internal/service`.
4. **Wire**: Chạy `make generate` để cập nhật Dependency Injection.
