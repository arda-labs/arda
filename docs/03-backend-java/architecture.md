## 📋 Overview

Core Banking Services (CBS) xử lý các nghiệp vụ tài chính quan trọng, yêu cầu độ chính xác cao và tính toàn vẹn dữ liệu tuyệt đối.

## 🏗️ Technical Stack

- **Framework**: Spring Boot 3.x
- **Language**: Java 21 / Kotlin
- **Persistence**: R2DBC (Reactive Relational Database Connectivity)
- **Native Image**: GraalVM Native Image
- **Database**: PostgreSQL 16+
- **Build Tool**: Gradle (Kotlin DSL)

    └── database/            # R2DBC configuration

## 🛡️ Key Patterns

### 1. Reactive Programming (Non-blocking)
Sử dụng **Project Reactor** (`Mono`, `Flux`) kết hợp với **R2DBC** để tối ưu hóa throughput.
- Phù hợp với các dịch vụ core có mật độ I/O cao.
- Tiết kiệm thread và memory so với JDBC truyền thống.

### 2. GraalVM Native Image Optimization
Để vận hành trong môi trường 32GB RAM, CBS được biên dịch sang mã máy:
- **Startup**: < 100ms.
- **RAM Usage**: ~50MB - 150MB (so với 500MB+ của JVM chuẩn).
- **AOT (Ahead-of-Time)**: Tất cả bean được khởi tạo lúc build time.

### 3. Saga Pattern for Distributed Transactions
Xử lý giao dịch phân tán giữa các Core services:
- Sử dụng **Orchestration Saga** (điều phối qua BPM Engine).
- Đảm bảo tính ACID trên toàn hệ thống thông qua các compensating transactions.

### 4. Database Migrations
Sử dụng **Flyway** (với JDBC driver riêng chỉ dùng cho migration) để quản lý version schema.

## 🚀 Native Build Workflow

1. **Development**: Chạy trên JVM chuẩn để debug nhanh.
2. **Analysis**: Sử dụng GraalVM Tracing Agent để thu thập reflection hints.
3. **Compile**: `./gradlew nativeCompile` để tạo binary runner.
4. **Test**: Chạy integration tests trên binary runner.
