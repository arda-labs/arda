## 🔄 CI/CD Flow

Quy trình triển khai của Arda dựa trên mô hình **GitOps**:

1.  **Code Change**: Developer push code lên `arda` repository.
2.  **CI (GitHub Actions)**:
    - Chạy Unit Test & Lint.
    - Build Docker Image (Sử dụng Buildx để tối ưu).
    - Push Image lên **GitHub Container Registry (GHCR)** với tag là `git-sha`.
    - **Quan trọng**: Action sẽ tự động gọi một script để update tag mới vào repo `arda-infra`.
3.  **CD (ArgoCD)**:
    - ArgoCD phát hiện thay đổi trong `arda-infra`.
    - Tự động "Sync" trạng thái mới xuống K3s cluster.

## 🔑 Secret Management

**Nguyên tắc**: Không bao giờ commit secret (mật khẩu, API key) lên Git dưới dạng plain text.

### Giải pháp đề xuất:
- **Bitnami Sealed Secrets**: Mã hóa secret thành một Custom Resource an toàn để commit lên Git. Chỉ có cluster mới giải mã được.
- **GitHub Secrets**: Dùng để chứa các thông tin nhạy cảm trong lúc Build (Docker login, SSH keys).

## 🛡️ Resource Management (Tối ưu cho 16GB/32GB RAM)

Mọi Deployment **BẮT BUỘC** phải có cấu hình `resources`:

```yaml
resources:
  requests:
    cpu: "100m"
    memory: "128Mi"
  limits:
    cpu: "500m"
    memory: "256Mi"
```

- **Requests**: Mức tài nguyên K8s cam kết cấp cho Pod.
- **Limits**: Mức tài nguyên tối đa Pod được phép dùng (vượt quá RAM sẽ bị OOMKill).

## 🏥 Health Checks

Để ArgoCD biết App có đang chạy tốt hay không:

- **Readiness Probe**: Kiểm tra App đã sẵn sàng nhận traffic chưa. (Nếu lỗi, K8s sẽ ngắt traffic tới Pod).
- **Liveness Probe**: Kiểm tra App còn sống không. (Nếu lỗi, K8s sẽ restart Pod).

Ví dụ cho Kratos (Go):
```yaml
readinessProbe:
  httpGet:
    path: /health
    port: 8000
livenessProbe:
  httpGet:
    path: /health
    port: 8000
```

## 📈 Monitoring & Alerts

- **Metrics**: Sử dụng Prometheus để thu thập số liệu từ `/metrics` endpoint của các service.
- **Visualization**: Grafana dashboard để theo dõi RAM/CPU của server Thinkcenter.
- **Alerts**: Gửi thông báo qua Telegram/Slack khi server sắp hết RAM hoặc Pod bị sập.
